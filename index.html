MCP 서버 연결하기

MCP 서버를 Claude Code에 추가하고, 연결을 확인하고,
디스크에서 설정 파일 위치를 찾는 방법을 알아보세요.

Model Context Protocol (MCP)을 사용하면 Claude Code가 이슈 트래커 검색, 데이터베이스 쿼리, 웹 브라우저 제어 등 기본 내장 도구 이상의 기능을 사용할 수 있습니다. 이러한 도구는 사용자의 머신 또는 호스팅 서비스로 실행되는 MCP 서버에서 제공됩니다.

이 가이드에서는 Claude Code CLI를 사용하여 하나의 MCP 서버를 처음부터 끝까지 연결하는 방법을 안내합니다. 완료 후에는 서버가 연결·응답 중인 상태가 되고, 설정 파일 위치와 가장 일반적인 연결 오류 해결 방법을 알 수 있습니다.

ℹ 데스크톱 앱, VS Code, 웹 등 다른 환경에서도 MCP 서버를 추가할 수 있습니다. 다른 환경에서 연결하기 섹션을 참고하세요.

시작하기 전에

다음 사항이 준비되어 있는지 확인하세요:

  • Claude Code 설치 및 인증 완료
  • 프로젝트 디렉터리에서 터미널 열기 (빈 디렉터리도 가능)

서버 추가 및 확인

아래 예시는 Claude Code 문서 MCP 서버에 연결합니다. 이 서버는 Claude Code 문서의 전체 텍스트 검색 기능을 제공하는 호스팅 서버로, 인증이나 별도 설정이 필요 없어 첫 번째 테스트 서버로 적합합니다.

01
MCP 서버 추가

서버를 Claude Code에 등록합니다. claude 세션 안이 아닌 터미널에서 실행하세요.

bash
claude mcp add --transport http claude-code-docs https://code.claude.com/docs/mcp

명령어 구성:

  • claude mcp add — 서버를 Claude Code에 등록
  • --transport http — 로컬 프로세스가 아닌 URL의 호스팅 서버
  • claude-code-docs — 직접 지정하는 이름 (다른 이름도 동일하게 동작)
  • https://code.claude.com/docs/mcp — 서버가 호스팅된 URL
02
연결 상태 확인

서버 목록에서 상태를 확인합니다:

bash
claude mcp list
상태 의미
✓ Connected 사용 준비 완료. claude-code-docs에서 표시되어야 합니다.
! Needs authentication 서버에 접근 가능하지만 브라우저 로그인 또는 토큰이 필요합니다.
✗ Failed to connect 서버가 응답하지 않습니다. 문제 해결 섹션을 참고하세요.
✗ Connection error 연결 시도 중 오류 발생. 문제 해결 섹션을 참고하세요.
⏸ Pending approval 아직 승인하지 않은 프로젝트 범위 서버.
03
서버 사용

세션을 시작하고 Claude에게 새 서버를 사용하도록 요청합니다:

bash
claude
프롬프트 예시
Use the claude-code-docs server to look up what MCP_TIMEOUT does
▸ 평소에는 프롬프트에 서버 이름을 명시할 필요가 없습니다. Claude가 자동으로 관련 도구를 선택합니다. 여기서는 데모를 위해 명시적으로 지정합니다.
04
서버 제거 (선택 사항)

실험이 끝나면 서버를 제거할 수 있습니다:

bash
claude mcp remove claude-code-docs
ℹ 연결된 각 서버의 도구 이름과 서버 지침이 모든 세션의 컨텍스트 창에 로드됩니다. 사용하지 않는 서버를 제거하면 컨텍스트 공간을 절약할 수 있습니다.

서버 저장 위치

claude mcp add 명령은 서버 세부 정보를 설정 파일에 저장합니다. 기본적으로 local 범위로 등록됩니다: 현재 프로젝트에서만 활성화되는 나만의 설정입니다.

local
나만의 설정. 현재 프로젝트에서만 활성화. 기본값.
~/.claude.json
project
프로젝트를 클론한 모든 팀원과 공유.
.mcp.json (프로젝트 루트)
user
나만의 설정. 모든 프로젝트에서 활성화.
~/.claude.json (mcpServers 키)

서버의 범위를 확인하려면:

bash
claude mcp get claude-code-docs

서버 범위 변경

서버를 추가할 때 범위가 고정됩니다. 범위를 변경하려면 기존 항목을 제거하고 새 범위로 다시 추가해야 합니다.

모든 프로젝트에서 서버 사용

user 범위로 추가하면 여는 모든 프로젝트에서 활성화됩니다 (나만 사용):

bash
claude mcp add --scope user --transport http claude-code-docs https://code.claude.com/docs/mcp

팀과 서버 공유

project 범위로 추가하면 프로젝트 루트의 .mcp.json에 저장됩니다:

bash
claude mcp add --scope project --transport http claude-code-docs https://code.claude.com/docs/mcp

.mcp.json을 버전 관리에 커밋하면 팀원들도 서버를 사용할 수 있습니다.

추가 MCP 서버 예시

로컬 서버 추가 — Playwright

로컬 stdio 서버는 Claude Code가 서브프로세스로 실행하는 프로그램입니다. Playwright MCP 서버는 Claude에게 브라우저를 제공하며, Node.js 18 이상이 필요합니다.

01
Playwright 서버 등록
bash
claude mcp add playwright -- npx -y @playwright/mcp@latest
02
연결 확인
bash
claude mcp list

첫 확인 시 npx가 패키지를 다운로드하는 동안 ✗ Failed to connect가 표시될 수 있습니다. 잠시 기다린 후 다시 실행하세요.

03
브라우저 사용
프롬프트 예시
Use playwright to open https://example.com and tell me the page title

OAuth 로그인 서버 연결 — Sentry

Sentry, Linear, Notion 같은 호스팅 서비스는 OAuth를 사용합니다. 서버 URL을 추가한 후 브라우저에서 로그인합니다.

01
서버 추가
bash
claude mcp add --transport http sentry https://mcp.sentry.dev/mcp
02
브라우저에서 인증

Claude Code 세션을 시작하고 MCP 패널을 엽니다:

명령어
/mcp

목록에서 sentry를 선택하고 Enter를 누른 후 Authenticate를 선택하세요.

03
서버 사용
프롬프트 예시
What Sentry projects do I have access to?

.mcp.json 직접 편집

프로젝트 루트에 .mcp.json을 직접 만들어 서버를 설정할 수도 있습니다. 이 파일은 리포지터리에 커밋되어 팀 설정으로 활용됩니다.

.mcp.json
{
  "mcpServers": {
    "claude-code-docs": {
      "type": "http",
      "url": "https://code.claude.com/docs/mcp"
    },
    "playwright": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@playwright/mcp@latest"]
    }
  }
}

파일 저장 후 새 Claude Code 세션을 시작하면 자동으로 읽어옵니다. 처음에는 서버 승인 프롬프트가 나타납니다.

다른 환경에서 연결

이 가이드에서는 claude mcp CLI 명령을 사용했지만 모든 Claude Code 환경에서 MCP 서버에 연결할 수 있습니다:

  • Claude Code 데스크톱 앱 — Connectors UI를 통해 서버 추가
  • Claude Desktop 채팅 앱 — claude mcp add-from-claude-desktop 명령으로 가져오기
  • VS Code — 외부 도구와 MCP 연결 문서 참고
  • 웹의 Claude Code — 리포지터리의 .mcp.json 읽기
  • Claude.ai — 계정에 추가한 커넥터가 CLI에서 자동으로 로드됨

문제 해결

서버가 연결되지 않으면 세션 내 /mcp 또는 셸에서 claude mcp list로 상태를 확인하세요.

/mcp에 "No MCP servers configured" 표시 ▼

Claude Code가 현재 디렉터리에서 서버를 찾지 못했습니다. 가장 일반적인 원인:

  • 다른 프로젝트에서 claude mcp add를 실행했습니다. 현재 프로젝트에서 다시 추가하거나 --scope user로 추가하세요.
  • 잘못된 경로의 설정 파일을 편집했습니다. 올바른 파일은 ~/.claude.json과 <project>/.mcp.json입니다.
상태가 "Failed to connect" 또는 "Connection error" ▼

HTTP 서버의 경우 URL이 접근 가능한지 확인하세요:

bash
curl -I https://mcp.sentry.dev/mcp

stdio 서버의 경우 터미널에서 직접 명령을 실행해 오류를 확인하세요:

bash
npx -y @playwright/mcp@latest
시작 시 연결 시간 초과 ▼

MCP_TIMEOUT 환경 변수로 제한을 늘리세요 (밀리초 단위):

bash
MCP_TIMEOUT=60000 claude
서버가 연결되었지만 도구가 표시되지 않음 ▼

세션 내 /mcp를 실행하고 서버를 선택해 도구 목록을 확인하세요. 목록이 비어 있다면 API 키와 같은 필수 환경 변수가 누락된 것입니다. --env KEY=value 옵션으로 변수를 전달하세요.

.mcp.json 변경 사항이 적용되지 않음 ▼

Claude Code는 세션 시작 시 .mcp.json을 읽습니다. 파일 편집 후 세션을 종료하고 다시 시작하세요. 이전에 서버를 거부한 경우:

bash
claude mcp reset-project-choices
OAuth 로그인 실패 또는 브라우저가 열리지 않음 ▼

/mcp를 실행하고 서버를 선택한 후 Authenticate를 다시 선택하세요. 브라우저가 자동으로 열리지 않으면 터미널에 표시된 URL을 복사해 수동으로 열어보세요.
